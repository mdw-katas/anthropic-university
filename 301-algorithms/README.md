# CS 301 — Design and Analysis of Algorithms

**Prerequisites:** CS 201
**Language:** language-agnostic (pseudocode; implement in your language of choice)

## Course Description

The design paradigms behind efficient algorithms — divide and conquer,
greedy, dynamic programming — and the analytical tools to prove them correct
and fast. Where CS 201 asked "what structure holds the data," this course
asks "what strategy solves the problem." Ends at the edge of intractability:
NP-completeness and what to do when no fast algorithm is known.

## Learning Objectives

- Prove an algorithm correct (loop invariants, exchange arguments,
  induction) and analyze its complexity (recurrences, the master theorem).
- Apply divide and conquer, greedy, and dynamic programming, and recognize
  which paradigm fits a new problem.
- Use the classic graph algorithms: shortest paths, minimum spanning trees,
  topological sort, strongly connected components.
- Explain P vs. NP, polynomial-time reductions, and NP-completeness.
- Reach for approximation, randomization, or heuristics when a problem is
  intractable.

## Topic Outline

1. Correctness proofs: loop invariants and induction.
2. Recurrences and the master theorem.
3. Divide and conquer: merge sort, quickselect, closest pair.
4. Greedy algorithms: interval scheduling, Huffman coding; exchange
   arguments.
5. Dynamic programming: memoization, tabulation; edit distance, knapsack,
   longest common subsequence.
6. Graph algorithms I: BFS/DFS applications, topological sort, strongly
   connected components.
7. Graph algorithms II: Dijkstra, Bellman-Ford, minimum spanning trees
   (Prim, Kruskal), union-find.
8. Network flow (survey): max-flow/min-cut and its surprising applications.
9. Intractability: P, NP, reductions, NP-completeness, the classic hard
   problems.
10. Coping with hardness: approximation algorithms, randomization,
    backtracking and branch-and-bound.
