# CS 101 — Introduction to Computer Science

**Prerequisites:** none
**Language:** Python

## Course Description

A first course in computer science: what computation is, how problems are
decomposed into algorithms, and how algorithms are expressed as programs.
Covers programming fundamentals through the lens of problem solving rather
than language syntax.

For an experienced engineer, the value here is vocabulary and formality — the
precise definitions (state, expression, side effect, invariant, complexity)
that later courses assume without ceremony. Expect to move quickly.

## Learning Objectives

By the end of this course you will be able to:

- Explain what an algorithm is and evaluate one informally for correctness
  and efficiency.
- Decompose a problem into functions with clear contracts.
- Trace program state by hand: variables, scope, the call stack, recursion.
- Use the core built-in data types (numbers, strings, lists, dictionaries)
  and choose appropriately among them.
- Reason about a program's growth rate informally (constant, linear,
  quadratic) as a bridge to formal analysis in CS 201/301.

## Topic Outline

1. What is computation? Algorithms, state machines, and the stored-program
   computer.
2. Values, types, expressions, and variables.
3. Control flow: conditionals, loops, and loop invariants.
4. Functions: contracts, parameters, scope, and the call stack.
5. Recursion: base cases, recursive cases, and the recursion/iteration
   duality.
6. Collections: sequences, mappings, and sets; iteration patterns.
7. Program organization: modules, testing your own code, debugging as
   hypothesis testing.
8. Informal efficiency: counting steps, why nested loops hurt, a first look
   at big-O intuition.
9. A tour of the field: where OS, networks, databases, languages, and AI fit,
   and how this degree visits each.
