Values, Types, Expressions, Variables

Four words you use daily. This lesson pins down what each one actually denotes, because the precise versions explain a whole family of bugs you have certainly met in the wild.

1. Values and types

A value is a piece of data: 42, "hello", True, 3.14. A type is a set of values plus the operations that are valid on them. int is "the whole numbers, with arithmetic"; str is "sequences of characters, with concatenation, slicing, searching". Types exist so that nonsense — "hello" / 3 — can be rejected instead of silently computed.

Python's core scalar types: int (arbitrary precision — no overflow, unlike every C-family language you've used), float (IEEE 754 double), bool (a subtype of int; True == 1), str (immutable Unicode), and None (the type NoneType's only value — "no answer").

2. Floats are binary fractions, not decimals

The most consequential fact in this lesson: 0.1 cannot be represented exactly in binary, for the same reason 1/3 cannot be written exactly in decimal. What's stored is the nearest 64-bit binary fraction. Hence the classic:

>>> 0.1 + 0.2 == 0.3
False

3. Expressions evaluate to values

An expression is anything that evaluates to a value: 2 + 3, len(name) > 8, price * (1 + tax_rate). A statement does something: assignment, if, return. The distinction matters because expressions compose — you can nest them indefinitely — and statements do not.

Two operators deserve special attention from an engineer raised on C-family semantics, because Python floors, C truncates:

>>> -7 // 2        # floor division: rounds toward -infinity
-4                  # (C would give -3)
>>> -7 % 2         # remainder takes the sign of the DIVISOR
1                   # (C would give -1)

Python guarantees (a // b) * b + (a % b) == a always. The practical payoff: x % n is always in 0..n-1, which is exactly what you want for wrapping indexes and hash buckets.

4. Variables are names bound to values

A Python variable is not a box holding bytes; it is a name bound to a value, like a label on a string tied to an object. Assignment rebinds the name; it never changes the value itself:

x = "hello"
y = x          # y now names the SAME value
x = x + "!"    # builds a NEW string, rebinds x to it
# y is still "hello" — nothing you did touched it

With immutable values (numbers, strings, tuples) this model is indistinguishable from copying. Where it bites — names sharing one mutable value — is the centerpiece of Lesson 6.

Predict: after a = 5; b = a; a = a * 2, what is b?

b is 5. b = a bound b to the value 5; rebinding a later is a's business alone.

5. Dynamic, but strong

Python is dynamically typed: names carry no type, values do, and checks happen at runtime. It is also strongly typed: values never change type behind your back, and "3" + 4 is a TypeError, not 7 (looking at you, JavaScript). Conversions are always explicit: int("3"), str(4), float(7).
