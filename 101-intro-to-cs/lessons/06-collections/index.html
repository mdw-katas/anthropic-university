Collections, Mutability, and Aliasing

Lesson 2 promised that "names bound to values" would eventually bite. This is the lesson where it bites — and where Python's four workhorse collections earn precise job descriptions.

1. The cast

All four are covered by the same half-open slicing and iteration idioms you already use. What's new here is not syntax — it is the semantics of sharing.

2. Aliasing: one value, many names

Assignment never copies. b = a binds a second name to the same value. For immutable values that's harmless. For mutable ones, a change through either name is visible through both — they are aliases:

Every "function mysteriously modified my list" bug is this diagram. Parameters alias their arguments: pass a list to a function, and the function's parameter is another name for your list. Mutating it mutates yours. This is also the full explanation of Lesson 4's mutable default trap: one list, created at def time, aliased by every call forever.

Predict: t = ([1, 2], "x") — does t[0].append(3) succeed?

Yes. The tuple is immutable: it will forever hold that same list and that same string. But the list it holds is still a list. Immutability is one level deep — a tuple is a fixed set of arrows, not a freezer.

3. Copies, shallow and deep

b = a.copy()            # new list, same element objects (shallow)
import copy
b = copy.deepcopy(a)    # recursively copies nested mutables too

Shallow is usually what you want and cheap; deep is for nested structures you intend to fork wholesale. Knowing which one a bug needs is a rite of passage.

4. Dicts and sets: the O(1) family

Dicts and sets are hash tables (CS 201 opens the hood). Practical consequences today: membership and lookup take roughly constant time — compared to a list's front-to-back scan — and their keys/elements must be immutable (hashable): strings, numbers, tuples yes; lists no.

counts = {}
for word in words:
    counts[word] = counts.get(word, 0) + 1   # the counting idiom

Comprehensions round out the toolkit — expression-shaped loops:

squares = [n * n for n in range(10)]
evens   = {n for n in numbers if n % 2 == 0}
by_id   = {user.id: user for user in users}
