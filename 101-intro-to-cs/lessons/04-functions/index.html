Functions, Contracts, and the Call Stack

A function is three things at once: a contract (what it promises, given what), a scope (a private namespace), and a stack frame (a live activation record). Professionals conflate the three daily and get away with it; this lesson takes them apart once, properly.

1. The contract view

A function's contract: if the caller supplies arguments meeting the precondition, the function delivers a result meeting the postcondition — and touches nothing else. That last clause has a name: a function with no observable effects beyond its return value is pure. Pure functions are trivially testable (you have been exploiting this in every assignment) and compose like Lego. In Python, the docstring states the contract:

def gcd(a, b):
    """Return the greatest common divisor of non-negative a and b.

    gcd(n, 0) == n by convention.        <- postcondition, edge included
    """

Also part of the contract, easy to forget: a Python function with no return — or a bare return — returns None. Silently. A whole genus of bugs (result = things.sort()) lives in that silence.

2. The scope view

Names created inside a function are local: born at call time, gone at return. Python resolves a name by searching scopes in LEGB order — Local, Enclosing function, Global (module), Built-in — and stops at the first hit. One rule with teeth: assigning to a name anywhere in a function makes it local everywhere in that function, even before the assignment line. Reading a global is free; shadowing it is automatic; writing it requires an explicit global declaration you should almost never use.

Predict: x = 10 at module level; a function does print(x) then x = 5. What happens at the print?

UnboundLocalError. The assignment on the later line made x local to the whole function, so the print refers to a local that doesn't exist yet. Scope is decided at compile time, not line by line.

3. The stack view

Each call pushes a frame — parameters, locals, and where to resume in the caller. Return pops it. That's the entire mechanism behind "where am I?" in every language you know, and it is why tracebacks look the way they do. Step through a short program:

def main():          # calls format_greeting, prints result
    message = format_greeting("Mike")
    print(message)

def format_greeting(name):   # calls shout, wraps result
    loud = shout(name)
    return "Hello, " + loud + "!"

def shout(text):
    return text.upper()

4. Parameters, defaults, and one famous trap

Arguments bind by position, by keyword (f(name="Mike")), or both. Default values are handy — and evaluated once, at definition time, not per call. With immutable defaults that's invisible. With a mutable default (def f(items=[])) it means every call shares one list that remembers previous calls. The idiom: default to None, create inside. Lesson 6 explains the machinery; for now, treat def f(x=[]) as a code smell with a wanted poster.
