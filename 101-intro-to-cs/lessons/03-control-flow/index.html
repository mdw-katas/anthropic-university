Control Flow and Loop Invariants

You could write Python's if, while, and for in your sleep. So this lesson spends its budget on the part you may never have been taught: how to reason about a loop instead of squinting at it — the loop invariant.

1. The mechanics, quickly

if n < 0:            # exactly one branch runs
    sign = -1
elif n == 0:
    sign = 0
else:
    sign = 1

while pending:        # repeat while the condition holds
    job = pending.pop()

for i in range(10):   # 0,1,...,9 — half-open, like slices
    print(i)

Points worth pinning down even for a veteran:

  • range(a, b) is half-open: includes a, excludes b. Half-open intervals compose without off-by-ones: range(0,5) + range(5,9) tile perfectly. Python is half-open everywhere (slices too), and once you internalize why, fencepost errors mostly vanish.
  • break exits the loop; continue skips to the next iteration; and Python's oddball: an else clause on a loop runs only if the loop finished without break — "else" as in "search failed".
  • Guard clauses (if bad: return early) beat nesting. You know this; now it has a name to hang on the syllabus.

2. The loop invariant: reasoning's power tool

A loop invariant is a statement about program state that is true before the loop starts and remains true after every iteration. Choose it well and correctness follows almost mechanically:

def largest(items):
    best = items[0]
    # INVARIANT: best == max of items[0..i], for the i seen so far
    for item in items[1:]:
        if item > best:
            best = item
    return best   # invariant + loop ended = best == max(items). QED.

Three checks, and they are exactly the structure of proof by induction (MATH 120 will make this formal): the invariant holds initially; each iteration preserves it; at exit, invariant + termination condition = the result you wanted.

3. Termination is not free

A loop is correct only if it also stops. The standard argument: find a quantity that strictly decreases every iteration and cannot decrease forever — loop bounds shrinking, b in Euclid marching toward zero. For Collatz, nobody has found one; that is precisely why it is an open problem and your Lesson 1 exploration carried a hidden asterisk.
