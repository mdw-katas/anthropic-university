# CS 490 — Senior Capstone Project

**Prerequisites:** senior standing (core complete; electives complete or
concurrent)
**Language:** your choice, justified in the proposal

## Course Description

A single substantial project that integrates the degree: proposed, designed,
built, tested, documented, and defended. The constraint that makes it a
capstone rather than a side project: it must draw on at least three courses
from the 300/400 level, include a written design document and a final
retrospective, and be developed in the open (versioned, tested, reviewed).

For this curriculum's student, the bar is deliberately above "another CRUD
service" — the project should be something professional work never gave you
an excuse to build.

## Learning Objectives

- Scope an ambitious project honestly: proposal, milestones, and risk
  assessment up front.
- Apply theory to practice: make design decisions traceable to principles
  learned in coursework, and document them.
- Sustain engineering discipline without external pressure: tests, CI,
  small commits, working software at every milestone.
- Evaluate the result: measure it, compare it to alternatives, and write a
  candid retrospective including what failed.
- Present and defend technical work in writing and in a final walkthrough.

## Structure

1. **Proposal** — problem statement, why it's capstone-worthy, which courses
   it exercises, milestones, risks.
2. **Design document** — architecture, key decisions with alternatives
   considered, testing strategy.
3. **Milestone demos** — working software at three checkpoints; scope
   renegotiated in writing if needed.
4. **Final deliverable** — the system, its tests, and user/operator
   documentation.
5. **Retrospective and defense** — measured results, honest failures,
   lessons; a recorded or written walkthrough.

## Example Directions (illustrative, not prescriptive)

- A relational database engine with a SQL subset, B+-tree storage, and WAL
  recovery (330 + 220 + 310).
- A compiled language with static types targeting a real ISA or WASM
  (410 + 340 + 370).
- A Raft-replicated key-value store verified with fault injection
  (420 + 310 + 320).
- A small LLM trained from scratch on a niche corpus, with an evaluation
  harness (440 + 430 + 360).
