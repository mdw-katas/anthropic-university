# CS 102 — Program Design and Abstraction

**Prerequisites:** CS 101
**Language:** Python

## Course Description

The second programming course: building larger programs by designing
abstractions. Object-oriented and functional decomposition, interfaces and
information hiding, error handling, and a first serious treatment of testing.
Ends with a small multi-module project.

You have practiced most of this professionally; the aim is to connect your
practice to the underlying principles and their names, so that CS 340
(Programming Languages) and CS 350 (Software Engineering) have a shared
foundation to build on.

## Learning Objectives

- Design programs around abstract data types: what an ADT is, and why the
  interface/implementation split matters.
- Apply object-oriented concepts precisely: encapsulation, inheritance vs.
  composition, polymorphism, dynamic dispatch.
- Use first-class functions: higher-order functions, closures, map/filter/
  reduce as design tools.
- Handle failure deliberately: exceptions, error values, and designing for
  partial failure.
- Write unit tests as executable specifications (arrange, act, assert).

## Topic Outline

1. Abstraction and specification: preconditions, postconditions, invariants.
2. Abstract data types; interface vs. implementation.
3. Classes and objects: encapsulation and representation invariants.
4. Inheritance, composition, and polymorphism; when each is the wrong tool.
5. First-class and higher-order functions; closures.
6. Error handling strategies: exceptions, sentinel values, result types.
7. Testing: unit tests, test doubles, designing for testability.
8. Iterators, generators, and lazy evaluation.
9. Building a multi-module program: dependency direction, cohesion,
   coupling.
10. Case study project: design, implement, test, and refactor a small
    system end to end.
