A Class Is a Fence Around an Invariant

Strip away the syntax and a class is three commitments: some data, the operations allowed to touch it, and a statement about that data — the representation invariant — that the constructor establishes, every method preserves, and no outsider can break. When you can say what a class's invariant is in one sentence, you understand the class. When you can't, neither did its author.

1. The representation invariant

class Fraction:
    """Rep invariant: _denominator > 0, and
    gcd(_numerator, _denominator) == 1 (lowest terms)."""

    def __init__(self, numerator, denominator):
        if denominator == 0:
            raise ValueError("denominator must be nonzero")
        if denominator < 0:
            numerator, denominator = -numerator, -denominator
        divisor = math.gcd(numerator, denominator)
        self._numerator = numerator // divisor
        self._denominator = denominator // divisor

Everything after the raise exists to establish the invariant. The payoff comes later: because every Fraction is in lowest terms with a positive denominator, equality can be a dumb field comparison and hashing can be a tuple hash. Methods get to assume the invariant on entry — that assumption is exactly Lesson 1's precondition, signed by the class with itself. Liskov calls checking it checkRep(); even unenforced, writing the sentence in the docstring sharpens the design.

2. Watch an invariant earn its keep (the widget)

With normalization off, 2/4 and 1/2 are the same number and unequal objects — field-comparison equality silently depends on the invariant. Every method that "assumes lowest terms" is one broken constructor away from lying.

3. Encapsulation in Python: conventions, not walls

Python has no private keyword. The community contract: a single leading underscore (_names) means "internal — touch it and the consequences are yours." A double leading underscore (__combination) triggers name mangling — the attribute is stored as _ClassName__combination — which exists to prevent accidental collisions in subclasses, not to keep anyone out; the mangled name is documented and reachable. The fence around your invariant is convention plus code review, and — as the explorations show — it is climbable on purpose and honored by the disciplined. Sixteen years of Go's lowercase-means-unexported should make this feel familiar, just less enforced.

4. The dunder contracts: __repr__, __eq__, __hash__

Objects that represent values (a fraction, a date, money) should compare by value, and that drags in a real contract:

Method Contract
__repr__ Unambiguous, ideally eval-able; for developers, not users
__eq__ Equivalence relation; return NotImplemented for foreign types
__hash__ a == b implies hash(a) == hash(b); only for immutable state

Two traps a veteran should have names for. First: define __eq__ and Python silently sets __hash__ to None — your objects stop working as dict keys unless you supply a hash. Second: without __eq__, equality is identity (is), so two structurally identical objects are unequal. Both are in the explorations; predict before you run.

Predict: class Team: members = [] — two Team instances each append one member. How long is Team.members?

Two. members lives on the class, so every instance shares the one list — attribute lookup falls through instance to class, and self.members.append(...) mutates the shared object without ever creating an instance attribute. Per-instance state belongs in __init__: self.members = []. (Assigning self.members = [...] would create an instance attribute — mutating and rebinding differ, as CS 101 Lesson 6 warned.)

5. Properties: computed attributes without breaking clients

class Fraction:
    @property
    def numerator(self):
        return self._numerator

@property lets attribute syntax run code, which means you can start with a plain attribute and later interpose validation or computation without editing a single caller. It is the abstraction barrier of Lesson 2 applied to attribute access — the representation stays negotiable even for things that look like fields.
